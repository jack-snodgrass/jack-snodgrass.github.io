ID vs Class CSS: Understanding the Differences

When using css to style the way your html will look we use selectors to target certain elements or a whole range of elements on the web page. Once we have selected the element a certain style, or even a set of styles can be applied to that element. There are actually a whole range of selectors that developers can use to style their html but the two most common ones and the ones I’ll be discussing are “ID” and “Class”. Most new developers will ask the questions what is the difference between the two? This is what ill be answering below:

ID:

IDs are the selector that developers will use to select and style a Single element on their webpage. Each web page can only have one element with a single ID attribute, and so the ID selector can never be used to style more than one element. IDs are indented by using the hashtag symbol followed by the name you are assigning.

Class:

Class is the selector that developers will use to select and style as many element that they want and that they apply the class tag too. As I have explained above , IDs are unique and can only be applied to one element on a webpage, so this is where the difference is. Classes can be assigned and use to style multiple elements. If you use a class to define a style this style will be applied to any and all elements that share that class name. Classes are indented by using a dot followed by the name of the class.

So in short, IDs are unique and can be applied to one element and Class is not unique and can be applied to multiple elements.
