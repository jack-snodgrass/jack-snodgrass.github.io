What Is Control Flow and Looping?

What is control flow?

Now that you are moving on from using just HTML and CSS and starting to build more intense, complex, and interactive web pages, mainly with using javascript, its impossible not to come across and start dealing with something known as “control flow”. This is the term used to describe flow or path that JavaScript takes when executing your code. The control part of that is the part of the code where we tell JavaScript to either "do this" or "do that" based on some condition.

There are four main types of control flow that we use:

  • Loops
  • If/else statements
  • Switch statements
  • Ternary operators

What are Loops?

Loops are used in Javascript to repeat a specific task based on a condition. The conditions we set out use a boolean to return a “true” or “false” value, and the loop will continue running until the defined condition returns “false”.

The three most common types of loops are:

  • For
  • While
  • Do While

Lets grab the first one "for" and have a bit more of a in depth look at it:

for ([initialization]); [condition]; [final-expression]) { // statement }

Above is and example of a for loop. The javascript for statement consists of three expressions and a statement:

  • initialization - Run before the first execution on the loop. This expression is commonly used to create counters. Variables created here are scoped to the loop. Once the loop has finished it’s execution they are destroyed.
  • condition - Expression that is checked prior to the execution of every iteration. If omitted, this expression evaluates to true. If it evaluates to true, the loop’s statement is executed. If it evaluates to false, the loop stops.
  • final-expression - Expression that is run after every iteration. Usually used to increment a counter. But it can be used to decrement a counter too.
  • statement - Code to be repeated in the loop

The DOM

The DOM, or Document Object Model, is a programme interface that defines the logical structure of a webpage. It represents the page so that programs can change the document structure, style, and content. It affects any languages you used (In our case at EDA its the HTML, CSS, and Javascript.)

Quick Recap on these:

HTML: This is what supplies the content to the webpage
CSS: This is a stylesheet which applies all the styling to a webpage
Javascript: Language that allows webpages to go from static to interactive.

What is shown in the browser when you load a page is a web document, its all the HTML, CSS, and JavaScript you have written coming together to form what you see and interact with on the page. Your browser read this and follows a specific pecking order to make sense of everything that’s going on. Your DOM is actually made up many kinds of things beyond just HTML elements. Everything that makes up your DOM is more generically known as nodes. These nodes can be elements (which shouldn't surprise you), attributes, text content, comments, document-related stuff, and various other things you simply never think about.
The DOM is the single most important piece of functionality you have for working with your HTML documents. It provides the missing link that ties your HTML and CSS with JavaScript. 

Explain the difference between accessing data from arrays and objects.

What is an object?

Objects are used when you want to store are collection of data, rather than just a signal value. They create a “something” which could really be anything eg: a person, an album, a book, anything that that is made up and/or defined by a group of data which are called “properties”. These properties consist of a key and a value, with the values being data types like strings, numbers, and booleans etc.

Below is a basic object of a person:

var person = {
name: 'Jack',
age: 23,
enjoyingEda: true };

We can return and manipulate the properties inside our objects by using either the dot or bracket notations. So if we wanted to access the object and return the value for our key “name” we could do so with the to notations like this:

/ Dot notation
person.age // returns 23
// Bracket notation
person['age'] // returns 23

If we wanted to add a property to our object to give the person a last name we could use the dot notation to do this as well, below is how we would do this then how our new object would look after the property is added:

person.lastName = ["Snodgrass"];
var person = {
name: 'Jack',
age: 23,
enjoyingEda: true;
lastName: ‘Snodgrass' };

What is an array?

An array, unlike and object, is used to store multiple items against a singular variable. These items can consist of strings, numbers, booleans, objects, or even other arrays, just as an object stores them against multiple keys and values. Items are accessed through arrays by using their numerical position. Arrays use what is called a zero-based indexing which means that that the first item in every array will be at position [0], and the second item being portion[1], and so on.

So if we had an array for favourite sports it would look like this:
var favouriteSports = ['Rugby', 'Basketball', 'MMA', ‘Surfing’]
If we wanted to return a value we use the variables name and the items numerical position:
favouriteSprorts[2]; - returns the value MMA

This will return the value ‘MMA’ as since we start from zero it is actually the second position inside the array. We can also add and remove items using a range of methods. The one below is the push method which adds and item on the end of the arry:

favouriteSports.push[‘Football’];
This will add Football onto the end of the array in the end position with the numerical value[4]:
var favouriteSports = ['Rugby', 'Basketball', 'MMA', ‘Surfing’, “Football”]

So Objects use the properties keys and call on these directly to access the data whereas arrays use the single variables name and the numerical position of the data they are trying to call.
